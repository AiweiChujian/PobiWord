PRD 06：记忆卡片会话

1. 场景定位

记忆卡片用于新词学习后或复习中，帮助用户通过随机顺序反复回忆单词。用户通过“忘记、模糊、认识”进行自评，系统据此更新单词在当前会话中的记忆记录。

2. 用户目标

  • 通过主动回忆巩固单词。
  • 在不知道时查看释义。
  • 快速标记已掌握单词，减少重复学习；必要时可以移除已掌握标记。

3. 入口

  • 新词学习卡片全部呈现完成后进入。
  • 复习单词会话中复用进入。
  • 首页“再次复习”入口进入。

4. 页面范围

  • 当前记忆进度。
  • 记忆卡片。
  • 发音入口。
  • 查看释义入口。
  • 忘记、模糊、认识三个自评按钮。
  • 标记为已掌握。
  • 移除已掌握标记。
  • 下一个单词按钮。
  • 会话完成页或返回首页。

5. 功能需求

编号功能说明优先级
F01随机展示单词按随机顺序展示记忆列表中的单词。P0
F02默认隐藏信息除单词本身外，其它信息不主动呈现。P0
F03播放发音用户可点击播放单词读音。P0
F04查看释义用户可主动展开释义、词性、例句等信息。P0
F05忘记单词记忆记录置为 0。P0
F06模糊不修改单词记忆记录。P0
F07认识单词记忆记录 +1。P0
F08释义后切换若释义已展示，自评后直接切换下一个单词。P0
F09先展示释义若释义未展示，自评后展示释义并呈现“下一个”按钮。P0
F10完成本轮记忆单词记忆记录达到 3 时，从记忆列表移除。P0
F11标记已掌握用户可将单词直接标记为已掌握，并从学习/记忆列表移除。P0
F12移除已掌握标记用户可以移除单词的已掌握标记，移除后该单词不再按已掌握词过滤。P1

6. 核心算法

6.1 自评处理

用户操作记忆记录处理后续展示
忘记置为 0若释义已展示，切换下一个；否则展示释义和“下一个”按钮。
模糊不处理若释义已展示，切换下一个；否则展示释义和“下一个”按钮。
认识+1若释义已展示，切换下一个；否则展示释义和“下一个”按钮。

6.2 下一个单词选择

  1. 如果记忆列表为空，则完成所有单词的记忆。
  2. 判断当前单词是否还在记忆列表中；在列表中则将其移除。
  3. 移除当前单词后，如果列表为空，则当前单词就是下一个需记忆单词。
  4. 从记忆列表剩余单词中随机挑选一个作为下一个需记忆单词。

7. 业务规则

  • 记忆记录是当前会话内的计数，达到 3 表示完成本轮记忆。
  • 标记已掌握后，该单词从当前学习列表和记忆列表移除。
  • 已掌握单词会在新的学习计划和后续选词中被筛掉。
  • 用户可以移除已掌握标记；移除后，该单词应按普通未掌握词参与后续计划和选词筛选。
  • 已掌握单词是 AI 生成例句时的重要参考。
  • 新词学习和复习都复用同一套记忆卡片交互。

8. 状态与异常

状态页面处理
记忆列表为空直接进入会话完成状态。
发音失败展示失败提示，支持重试。
释义内容缺失展示内容缺失提示并提供重新生成或跳过。
用户中途退出保存会话状态，返回后可继续。
标记已掌握误触用户可移除已掌握标记，恢复为未掌握状态。

9. 数据需求

  • 会话 ID、计划 ID、批次 ID。
  • 记忆列表单词 ID。
  • 当前单词 ID。
  • 当前单词记忆记录。
  • 释义是否已展示。
  • 已掌握状态。
  • 已掌握标记更新时间与移除状态。
  • 发音、释义、例句等学习卡片内容。

11. 验收标准

  • 三个自评按钮按规则正确更新记忆记录。
  • 单词记忆记录达到 3 后从记忆列表移除。
  • 记忆列表为空时会话完成。
  • 标记已掌握会同步影响当前会话与后续学习计划筛选。
  • 移除已掌握标记后，单词不再被已掌握词过滤规则排除。
