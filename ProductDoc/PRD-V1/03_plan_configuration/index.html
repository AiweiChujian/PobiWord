PRD 03：学习计划配置

1. 场景定位

用户选择学习来源后，需要配置每日学习节奏。系统基于剔除已掌握词后的待学习单词，计算学习压力和预计完成日期。每日提醒为全局设置项，计划配置页仅提供跳转到 Profile 设置中开启或调整提醒的入口。

2. 用户目标

  • 知道本计划实际需要学习多少单词。
  • 设置每次新单词学习量。
  • 了解计划压力和预计完成时间。
  • 按需跳转到设置中开启或调整每日学习提醒。

3. 入口

  • 学习计划来源选择页确认后进入。
  • 后续如支持编辑计划，可从计划设置页进入。

4. 页面范围

  • 展示来源信息与过滤后的待学词量。
  • 配置每次新单词学习量。
  • 展示最大学习压力值预估。
  • 展示预计完成日期。
  • 每日提醒设置入口。
  • 完成计划创建。

5. 功能需求

编号功能说明优先级
F01已掌握词过滤基于用户当前已掌握词集合剔除来源词表中的已掌握词。P0
F02待学词量展示展示过滤后的计划学习词数。P0
F03新词学习量配置默认值 30，用户可调整。P0
F04学习压力预估最大学习压力值 = 每次新词学习量 × 5。P0
F05预计完成日期展示新词完成学习日期与最后复习完成日期。P0
F06复习完成日期计算最后复习完成日期 = 新词完成后第 13 天。P0
F07每日提醒设置入口展示每日提醒当前状态，并提供跳转到 Profile 设置页开启提醒或设置提醒时间的入口。P1
F08完成创建保存计划并回到首页/计划页。P0

6. 业务规则

  • 默认每次新单词学习量为 30。
  • 每次新词学习量必须大于 0。
  • 最大学习压力值按新词学习量的 5 倍展示，用于让用户理解学习负担。
  • 新词完成学习日期基于待学词量和每次新词学习量计算。
  • 最后复习完成日期为新词完成日期后 13 天。
  • 每日提醒是全局设置，每日仅提醒一次；提醒开关和提醒时间在 Profile 设置页维护。
  • 计划配置页不直接保存提醒时间，只展示入口和当前提醒状态。
  • 用户移除某个单词的已掌握标记后，该单词不再从待学词表中过滤。
  • 若过滤后待学词量为 0，应阻止创建计划，并提示该来源内单词已全部掌握或无可学习单词。

7. 状态与异常

状态页面处理
待学词量为 0展示说明，引导返回更换单词书。
新词学习量为空或非法禁用完成按钮并提示输入有效数字。
通知权限未开启创建计划可继续；点击提醒设置入口时引导到 Profile 设置页完成授权和提醒时间设置。
保存失败展示失败提示，保留当前配置并支持重试。

8. 数据需求

  • 来源 ID、来源类型、来源名称。
  • 原始词表数量。
  • 当前已掌握词数量。
  • 过滤后待学词表。
  • 每次新词学习量。
  • 学习压力预估值。
  • 新词完成日期、最后复习完成日期。
  • 每日提醒开启状态与提醒时间，仅用于展示和跳转入口。

9. 埋点

  • plan_config_page_view
  • plan_new_word_count_change
  • plan_reminder_settings_click
  • plan_create_submit
  • plan_create_success
  • plan_create_fail

10. 验收标准

  • 用户进入页面后可看到过滤后的待学习词量。
  • 修改新词学习量后，压力预估和预计完成日期同步更新。
  • 每日提醒入口可跳转到 Profile 设置页；计划配置页不直接保存提醒时间。
  • 待学词量为 0 或配置非法时无法创建计划。
  • 创建成功后首页能正确展示新计划状态。
