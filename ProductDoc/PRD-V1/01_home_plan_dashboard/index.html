PRD 01：首页与计划状态

1. 场景定位

首页是用户进入 Pobi 后的主场景，也是其它 V1 功能的分发入口。V1 允许同时存在多个学习计划，但首页只展示最近选中的计划，并根据该计划状态展示创建/新增学习计划、选择其它计划、学习新单词、复习、再次复习或完成统计等直接入口；部分入口会间接进入单词书选择、计划配置、新词批次选择、学习卡片和记忆卡片等后续页面。

2. 用户目标

  • 一眼知道今天应该做什么，以及下一步会进入哪个学习流程。
  • 在有复习任务时优先完成复习。
  • 在复习完成后继续学习新词或再次复习。
  • 在多个计划之间切换，并让首页展示最近选中的计划。
  • 在无计划或允许新增计划时进入创建/新增学习计划流程。
  • 在计划完成后查看结果。

3. 入口

  • APP 启动后默认进入。
  • 底部 Tab 或主导航中的首页入口。
  • 创建学习计划完成后返回。
  • 学习、复习、记忆会话完成后返回。

4. 页面范围

  • 学习计划卡片。
  • 当前选中计划展示。
  • 计划选择/切换入口。
  • 今日操作入口。
  • 创建/新增学习计划入口。
  • 学习、复习、完成统计等状态入口。
  • 学习进度概览。
  • 计划完成状态入口或统计占位视图。

5. 功能需求

编号功能说明优先级
F01无计划状态用户无计划时展示创建学习计划入口，点击后进入学习计划来源选择。P0
F02计划基础信息展示计划名称、来源、待学进度、复习进度。P0
F03新计划入口状态无学习记录的计划仅展示“学习新单词”入口。P0
F04复习优先状态每次学习新单词之后，首页先只展示复习入口。P0
F05复习完成状态完成复习后，同时展示“再次复习”和“学习新单词”入口。P0
F06新词学完状态所有单词已添加学习后，仅展示复习入口。P0
F07计划完成状态所有单词复习完成后，展示学习计划完成状态；统计页面和内容待定时展示占位视图。P0
F08多计划展示允许同时存在多个学习计划；首页只展示最近选中的计划，并提供计划选择和新增计划入口。P0
F09入口分发根据计划状态，将用户分发到创建计划、学习新词、复习、再次复习或完成统计。P0

7. 首页入口映射

首页入口目标页面/流程入口类型
创建/新增学习计划学习计划来源选择 → 学习计划配置直接入口，后续间接进入配置流程
选择其它计划计划选择列表 → 首页刷新为选中计划状态直接入口，切换后记录最近选中的计划
学习新单词新词批次选择 → 新词学习卡片 → 记忆卡片会话直接入口，后续间接进入学习与记忆流程

8. 业务规则

  • 在学习新单词之前必须先完成复习。
  • V1 允许同时存在多个学习计划；首页只展示最近选中的计划。
  • 用户在首页切换计划后，该计划成为最近选中的计划，首页所有状态和入口都基于该计划计算。
  • 新增计划创建完成后，默认将新计划设为最近选中的计划并返回首页。
  • 首页入口由计划进度、学习序号、复习任务和新词剩余量共同决定。
  • “再次复习”不改变计划学习序号。
  • 计划完成后不再展示学习和复习入口，改为展示完成状态/统计占位。

9. 状态与异常

状态页面处理
计划数据加载中展示加载状态。
计划数据加载失败展示重试入口。
通知权限未开启可在计划卡片展示提醒授权提示。
多端数据冲突以服务端最新计划状态为准，并刷新入口。

10. 数据需求

  • 计划 ID、计划名称、来源信息。
  • 计划列表、最近选中计划 ID。
  • 总待学词数、已加入学习词数、已掌握词数。
  • 当前学习序号。
  • 今日/当前轮复习任务状态。
  • 是否存在未学习新词。
  • 是否所有复习均已完成。

12. 验收标准

  • 首页能根据不同计划状态展示正确入口。
  • 存在多个学习计划时，首页只展示最近选中的计划。
  • 用户可以在首页切换计划，切换后计划状态和操作入口同步刷新。
  • 存在待复习任务时，用户无法绕过复习直接学习新词。
  • 所有新词已学习后，不再展示新词学习入口。
  • 计划完成后展示完成状态；统计页面和内容待定时展示占位视图。
