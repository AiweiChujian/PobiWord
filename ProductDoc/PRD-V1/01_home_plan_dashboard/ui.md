# 01 首页与学习计划状态 UI 说明

> 文档用途：作为 AI 设计工具生成线框图、交互原型、Figma 视觉稿和开发实现的统一依据。  
> 对应需求：`PRD-V1/01_home_plan_dashboard/index.html` 与本轮补充需求。  
> 目标平台：iOS 手机端；基准画板 390 × 844 pt，适配 375–430 pt 宽度。  
> 锁定视觉基准：`wireframe-prototype/qa-artifacts/h-03a-visual-direction-final.png`。H-03A 为 H-02、H-03B 的共享布局与 UI 语言来源。

## 1. 页面目标

首页是 Pobi 的默认启动页，用于呈现当前学习计划、学习阶段和下一步任务。用户进入首页后，应能立即判断：

1. 是否已有学习计划。
2. 当前查看的是哪个计划，计划总体进度如何。
3. 当前应当“学习”还是“复习”。
4. 当前主任务包含多少单词，点击哪里开始。
5. 是否存在计划外的自由巩固入口。
6. 如何切换/创建计划，以及进入 Profile 和设置。

首页遵循“单一主任务”原则：一个状态只突出一个主入口。计划统计是辅助信息，Activity 是历史反馈，二者均不得抢占主入口的视觉重心。

## 2. 数据与状态边界

- 首页所有状态和数据均来自 APP 本地数据库。
- 本场景不设计“加载中”“加载失败”“完成”状态，也不显示加载占位、错误页、重试或完成庆祝页。
- 首页业务画板：

| 画板编号 | 状态 | 核心差异 |
| --- | --- | --- |
| H-01 | 无学习计划 | 创建计划空状态 |
| H-02 | 有计划、无学习记录 | 学习主题主入口；不显示 Activity |
| H-03A | 有记录、当前应复习 | 复习主题主入口；显示 Activity |
| H-03B | 有记录、当前应学习 | 学习主题主入口；显示 Activity |
| O-01 | 学习计划下拉菜单 | 覆盖在 H-02/H-03 上方 |
| O-02 | Profile + 设置侧边栏抽屉 | 可覆盖任意首页状态 |

H-02、H-03A、H-03B 使用同一页面骨架、同一仪表盘组件和同一主入口组件，仅通过状态数据切换文案、颜色、图标、按钮和 Activity 是否出现。

## 3. 有计划状态的固定信息顺序

H-02、H-03A、H-03B 从上到下固定为：

1. 系统状态栏。
2. NavigationBar。
3. `PlanDashboardSection`：计划统计仪表盘。
4. `StageEntrySection`：当前学习阶段主入口。
5. `ActivityHeatmapSection`：近 60 天学习活动；仅 H-03A、H-03B 显示。
6. 底部安全区。

不在仪表盘上方增加 Header；不增加“计划统计”“查看全部”等标题或操作。不在 Activity 上方增加标题，不显示图例。

## 4. 全局视觉方向

### 4.1 设计语言

- 当前锁定视觉方向采用深色页面背景、系统大标题排版、清晰的 Secondary / Tertiary Background 层级。
- 整体简约，局部通过高饱和渐变、渐变进度环和专属任务图标体现 Pobi 的识别度。
- 框架、导航、浮层、按钮反馈、圆角和动效优先采用 UIKit / SwiftUI 系统习惯。
- 图标使用 SF Symbols 或同风格图标库；不使用 Emoji、文本符号、手绘占位图或不一致的图标风格。
- 模块之间主要依靠背景层级、圆角和留白区分，不使用霓虹发光、渐变描边或商业海报式 Banner。

### 4.2 栅格与尺寸原则

- 页面水平安全边距：20 pt。
- NavigationBar 与首个 Section 之间保持紧凑但清晰的间距。
- Section 纵向间距：12–16 pt；同一 Section 内间距优先压缩，确保首屏同时出现仪表盘、入口和 Activity 的主要内容。
- Section 圆角：22–28 pt；三级信息卡圆角：12–16 pt。
- 主入口 Section 是页面最高、色彩最强的模块，但不做海报式大标题，也不通过无意义的上下留白增加高度。
- 所有可点击控件的命中区域至少 44 × 44 pt；视觉图标可以更小。

### 4.3 色彩语义

- 页面背景：System Background 的深色值。
- 仪表盘和 Activity：Secondary Background Color。
- 仪表盘右侧 Item：各自的 Tertiary Background Color，以低对比度淡色承载信息。
- 复习主题：从左下到右上的蓝色 → 紫色 → 品红色渐变。
- 学习主题：从左下到右上的鲜活绿色 → 黄绿色 → 黄色渐变。
- 进度环：起始为绿色，当前进度末端过渡到浅黄色；未完成轨道使用低对比度灰色。
- 待学习：偏红主题；已掌握：蓝紫主题；预计完成日期：橙色主题。
- Activity：灰色、暗绿、浅绿、亮绿四级离散状态，不使用连续透明度替代语义。

## 5. 全局组件

### 5.1 NavigationBar

- 左上角始终显示侧边栏按钮，点击打开 O-02；图标建议使用 `line.3.horizontal`。
- H-01 中间标题为“Pobi”，不可点击。
- H-02、H-03A、H-03B 中间标题为当前计划 `title`，右侧显示向下箭头；标题与箭头整体点击打开 O-01。
- 标题单行显示，超长时省略；右侧默认留空，保持视觉平衡。
- 有计划状态的菜单按钮使用圆形低对比度背景，与深色页面融合。

### 5.2 计划统计仪表盘 `PlanDashboardSection`

#### 整体

- 无 Header，无“查看全部”。
- 使用 Secondary Background Color 和 Section 圆角。
- 固定为左侧圆形进度、右侧三个统计 Item 的横向结构。
- Section 高度保持紧凑；通过缩小内容高度、上下内边距和 Item 垂直间距压低整体高度。
- 内容与 Section 左右边缘保持舒适留白；右侧 Item 组宽度小于左侧进度区与其间空白的总视觉宽度。

#### 左侧进度

- 使用宽圆环，环宽明显高于系统默认细进度环。
- 完成轨道使用绿色到浅黄色的渐变，渐变末端对应当前进度位置。
- 圆环中央两行错位排版：
  - 第一行 `learnedCount`，居中、大号、主文字颜色。
  - 第二行 `/{totalCount}`，换行、浅色小字体。
- 圆环下方显示“已加入学习”，作为次要标签。
- 示例：第一行 `360`，第二行 `/1200`。

#### 右侧 Item

- 上二下一排列：上方并排“待学习”“已掌握”，下方横跨两列“预计完成日期”。
- 每个 Item 有自己的低对比度三级背景色和圆角，内容均左对齐。
- Item 图标尺寸与标题字体等高，不得大于标题行高。
- 标题为次要文字；数值使用对应主题色，但字号应克制，低于圆环主数字。
- 待学习：偏红色；已掌握：蓝紫色；预计完成日期：橙色。
- 日期格式固定为 `yyyy-MM-dd`，例如 `2026-10-01`。
- 若预计完成日期不存在，隐藏下方 Item，并让上方 Item 组自然居中；不显示破折号或空卡片。

### 5.3 当前阶段入口 `StageEntrySection`

#### 共同结构

- 整个 Section 是一张实心渐变背景卡片，不使用渐变边框和霓虹发光。
- 左侧内容从上到下：阶段标题、单词数量、简短说明、主按钮、可选次入口。
- 右侧为状态图标；图标容器必须是正圆，不得因布局压缩为椭圆。
- 卡片不做 Banner/海报式排版，不使用多个大标题；阶段标题克制，单词数量是主要数字。
- 不通过增大标题、说明、按钮之间的垂直间距拉高 Section；内容应紧凑居中。
- 主按钮使用高对比度胶囊/圆角按钮；次入口使用更轻的文字按钮或透明材质按钮。

#### 复习主题

- 背景：左下蓝色，经紫色到右上品红色。
- 标题：“本轮复习”。
- 图标：由“单词卡片 + 循环箭头”组合表达复习；使用 SF Symbols 或同风格标准图标，不绘制不协调的外边框。
- 图标容器：低透明度材质圆 + 细高光边缘，使其在渐变背景上保持协调。

#### 学习主题

- 背景：左下绿色，经黄绿色到右上黄色；色彩比仪表盘进度更鲜活。
- 标题、数字、单位、说明、次入口和学习图标统一使用白色前景，与 H-03A 保持一致；说明文字可使用白色 88% 透明度形成层级。
- 主按钮与 H-03A 使用相同的白色实心按钮体系，按钮文字改用学习主题绿色；不使用黑色或深色按钮背景。
- 标题：“本轮学习”。
- 图标：由“打开的书/单词卡 + 闪光”组合表达学习和新内容；使用 SF Symbols 或同风格标准图标。
- 图标容器沿用复习主题的正圆材质系统，使用白色图标、半透明材质背景和轻微白色高光边缘。

### 5.4 近 60 天活动 `ActivityHeatmapSection`

- 仅 H-03A、H-03B 显示；H-02 不显示，也不预留空位。
- 使用 Secondary Background Color 和 Section 圆角。
- 不显示“Activity”标题，不显示图例。
- 固定 12 列 × 5 行，共 60 个方格；方格之间使用小间隙，保证热力图紧凑。
- 右下角显示“近 60 天”小字，并减少它与 Section 底部的间距，控制 Section 总高度。
- 四级状态：

| Level | 颜色 | 含义 |
| --- | --- | --- |
| 0 | 灰色 | 未学习 |
| 1 | 暗绿 | 仅完成复习 |
| 2 | 浅绿 | 完成复习和学习 |
| 3 | 亮绿 | 完成复习和学习，并完成强化/巩固 |

- 单日格可点击时进入对应日期详情；V1 原型中允许只提供辅助说明，不新增详情页。

### 5.5 无计划空状态 `EmptyPlanState`

- 仅 H-01 显示。
- 使用与 H-02/H-03 相同的深色 System Background 和 NavigationBar，但空状态内容直接置于页面背景上，不使用 Section、Secondary Background、边框或卡片容器。
- NavigationBar 中间标题“Pobi”。
- 不显示“开始学习”等眉题。内容只包含学习主题绿色—黄色圆形图标、标题、简短说明和“创建学习计划”主按钮。
- 学习图标沿用 H-02/H-03B 的“打开的书/单词卡 + 闪光”组合；主按钮使用白色实心背景和学习主题绿色文字。
- 不显示空仪表盘、空 Activity 或禁用学习入口。

## 6. 分状态规格

### 6.1 H-01：首页 / 无学习计划

触发条件：本地数据库中没有学习计划。

- 标题：“创建你的单词学习计划”。
- 说明：“选择或导入一本单词书，开启单词学习之旅。”
- 主按钮：“创建学习计划”；点击进入学习计划来源选择流程。
- 左上角菜单仍可打开 O-02。

### 6.2 H-02：首页 / 有计划、无学习记录

触发条件：计划已创建，但没有任何学习或复习记录。

页面顺序：NavigationBar → 仪表盘 → 学习入口。没有 Activity。

示例数据：

- 当前计划：“雅思核心词汇”。
- 已加入学习：`0 / 3000`。
- 待学习：`3000`。
- 已掌握：`0`。
- 预计完成日期：`2026-12-18`。

入口内容：

- 主题：学习渐变（绿色 → 黄色）。
- 标题：“本轮学习”。
- 数量：“30 个单词”。
- 说明：“完成后，Pobi 会安排复习”。
- 图标：学习图标（打开的书/单词卡 + 闪光）。
- 主按钮：“开始学习”；点击进入“新词批次选择”。
- 不显示次入口，不显示 Activity。

### 6.3 H-03A：首页 / 有记录、当前应复习

触发条件：当前计划有学习记录，且本轮存在系统安排的待复习单词。

页面顺序：NavigationBar → 仪表盘 → 复习入口 → Activity。

示例数据：

- 当前计划：“CET-4 核心词汇”。
- 已加入学习：`360 / 1200`。
- 待学习：`840`。
- 已掌握：`180`。
- 预计完成日期：`2026-10-01`。

入口内容：

- 主题：复习渐变（左下蓝色 → 紫色 → 右上品红色）。
- 标题：“本轮复习”。
- 数量：“42 个单词”。
- 说明：“先完成复习，再继续计划”。
- 图标：复习图标（单词卡片 + 循环箭头）。
- 主按钮：“开始复习”；点击进入单词复习流程。
- 次入口：“巩固学习”；点击进入上一轮新学单词的自由练习。
- 次入口属于计划外自由选择，不改变计划进度；无可用词表时隐藏。
- 显示近 60 天 Activity。

### 6.4 H-03B：首页 / 有记录、当前应学习

触发条件：当前计划有学习记录，本轮计划复习已完成，当前应继续学习新词。

页面顺序：NavigationBar → 仪表盘 → 学习入口 → Activity。

示例数据沿用当前计划的本地统计：`360 / 1200`、待学习 `840`、已掌握 `180`、预计完成 `2026-10-01`。

入口内容：

- 主题：与 H-02 相同的学习渐变（绿色 → 黄色）。
- 标题：“本轮学习”。
- 数量：“30 个单词”。
- 说明：“复习已完成，继续推进计划”。
- 图标：与 H-02 相同的学习图标系统（打开的书/单词卡 + 闪光）。
- 主按钮：“开始学习”；点击进入“新词批次选择”。
- 次入口：“强化复习”；点击进入上一轮复习单词的自由练习。
- 次入口属于计划外自由选择，不改变计划进度；无可用词表时隐藏。
- 显示近 60 天 Activity。

## 7. 浮层交互

### 7.1 O-01 学习计划下拉菜单

- 点击有计划状态的 NavigationBar 标题打开。
- 以标题下方的系统菜单、Popover 或 Bottom Sheet 形式呈现；使用 UIKit / SwiftUI 系统材质和圆角。
- 包含当前计划、其他计划和“创建新计划”。当前计划显示勾选状态。
- 点击计划后切换 NavigationBar、仪表盘、入口和 Activity 数据；点击“创建新计划”进入创建流程。
- 点击遮罩、再次点击标题或下拉后按系统返回关闭。

### 7.2 O-02 Profile + 设置侧边栏抽屉

- 点击左上角按钮后从左侧推出，背景加系统遮罩。
- 内容包括 Profile 摘要、学习额度、提醒设置、声音与播放、更多设置、隐私政策、用户协议和关于 Pobi。
- 支持点击遮罩、关闭按钮或右滑关闭；首页状态不变。

## 8. 本地数据绑定

建议的数据字段：

```text
HomeState
- activePlan: Plan?
- hasStudyRecords: Bool
- currentStage: none | review | learn
- activityHistory: [ActivityDay]

Plan
- id: String
- title: String
- learnedCount: Int
- totalCount: Int
- remainingCount: Int
- masteredCount: Int
- currentBatchCount: Int
- currentReviewCount: Int
- estimatedCompletionDate: Date?
- optionalPracticeAvailable: Bool

ActivityDay
- date: Date
- level: 0 | 1 | 2 | 3
```

派生规则：

- `activePlan == nil` → H-01。
- `activePlan != nil && hasStudyRecords == false` → H-02。
- `hasStudyRecords == true && currentStage == review` → H-03A。
- `hasStudyRecords == true && currentStage == learn` → H-03B。
- `activityHistory` 只读本地最近 60 天；不足 60 天的日期按 Level 0 补齐。
- 日期在仪表盘中格式化为 `yyyy-MM-dd`。

## 9. 动效与反馈

- 页面状态切换、计划切换和抽屉使用 iOS 系统弹簧或标准缓动，不使用持续霓虹、漂浮或装饰性粒子。
- 点击主入口提供轻微缩放和触觉反馈，随后进入对应流程。
- Activity 格和仪表盘 Item 仅使用轻微按压反馈，不抢夺主入口注意力。
- Reduce Motion 开启时取消位移和缩放，仅保留淡入淡出。

## 10. 无障碍与设计验收

- 颜色不是唯一区分方式：任务类型同时通过标题、按钮文案和图标表达。
- 主按钮、菜单按钮、计划标题、次入口命中区域不小于 44 × 44 pt。
- 深色背景上的正文、渐变卡片上的按钮文字满足 WCAG AA 对比度。
- Dynamic Type 放大后，圆环主数字与总数仍保持两行，不相互覆盖；右侧日期可缩放但不截断。
- 学习图标与复习图标需要语义可辨，且图标容器在任何宽度下保持 1:1。
- H-02 不出现 Activity；H-03A/H-03B 的 Activity 必须为 12 × 5，共 60 格。
- H-02/H-03B 共享学习渐变和学习图标；H-03A 使用复习渐变和复习图标。
- H-02/H-03A/H-03B 的仪表盘布局完全一致，只替换数据。
- 页面不出现加载、失败、重试、完成状态或其占位 UI。

## 11. 原型交付要求

- 原型需提供 `?state=empty`、`?state=new`、`?state=review`、`?state=learn` 直接访问状态。
- 左上角侧边栏、计划标题下拉、主入口、可用的次入口均可交互。
- 原型控制面板仅用于评审，不属于正式 APP UI。
- H-03A 的实现必须以锁定视觉方向图为主要视觉基准；H-02 和 H-03B 从该基准派生，不另起布局或视觉体系。
