Pobi APP V1 场景与 PRD 总览

1. 文档来源

  • 来源流程文档：ProductDoc/pobi_main_flow.md
  • 本目录将主流程拆分为独立场景/页面 PRD，便于后续进入交互设计、研发拆解和测试验收。

2. 场景划分原则

  • 以用户目标为主：每个 PRD 对应一个清晰的用户任务或业务状态。
  • 页面与流程分离：可复用能力（如记忆卡片、购买额度）独立成文档，避免在多个页面重复描述。
  • 聚焦 V1 范围：仅保留当前主流程中可进入首版实现的页面与能力。

4. 关键对象

对象说明
单词书创建学习计划的基础词库来源。
学习计划绑定一个词库来源、计划配置、学习序号和学习进度。V1 允许同时存在多个计划，首页默认展示最近选中的计划。
学习序号从 1 开始，用于计算新词学习轮次和复习周期。
新词批次每次进入新词学习时选定的一组待学单词。
记忆记录单词在一次记忆会话内的计数，达到 3 后完成本轮记忆。
已掌握单词从学习和记忆列表中剔除，同时作为 AI 例句生成的重要参考。用户可以移除已掌握标记。
学习额度按单词数计费，用户可添加学习的单词额度，选词时校验。
Profile承接剩余额度查看、每日提醒设置、APP 设置和其它 APP 信息。

5. V1 主流程

主入口 首页与计划状态 根据计划状态分发到创建、学习、复习或完成统计

无计划 / 新增计划

  1. 01选择单词书
  2. 02配置学习计划
  3. 03返回首页并选中该计划

存在待复习词

  1. 01进入复习单词会话
  2. 02复用记忆卡片完成复习
  3. 03更新学习序号与进度
  4. 04返回首页判断下一步

无待复习且有新词

  1. 01选择新词批次
  2. 02学习卡片逐词呈现
  3. 03进入记忆卡片会话
  4. 04更新学习序号与进度

新词与复习均完成

  1. 01展示学习计划完成统计
  2. 02用户可创建新的学习计划

6. V1 已确认决策

  • V1 允许同时存在多个学习计划；首页只显示最近选中的计划，并提供选择其它计划和新增计划入口。
  • 学习额度的计费单位为单词数；本轮预计消耗额度等于本轮选择的单词数。
  • 已掌握单词支持移除已掌握标记；移除后应按普通未掌握单词参与后续筛选。
  • 学习计划完成统计页面和具体统计内容待定；V1 仅确认完成状态入口和占位能力。
  • 每日仅提醒一次；提醒开关和提醒时间在 Profile 设置中维护，计划配置页只提供跳转设置入口。
