PRD 10：Profile 与设置

1. 场景定位

Profile 是 Pobi 的个人与设置页面，用于查看用户剩余学习额度、管理每日提醒和基础 APP 设置，并展示版本、协议、隐私等 APP 信息。

2. 用户目标

  • 查看当前剩余学习额度。
  • 进入购买或补充学习额度流程。
  • 开启/关闭每日提醒，并设置唯一的每日提醒时间。
  • 查看 APP 版本、协议、隐私政策和其它基础信息。

3. 入口

  • 底部 Tab 或主导航中的 Profile 入口。
  • 计划配置页的每日提醒设置入口。
  • 学习额度不足提醒中的购买或查看额度入口。

4. 页面范围

  • 用户信息区域。
  • 剩余学习额度展示。
  • 购买/补充学习额度入口。
  • 每日提醒开关与提醒时间设置。
  • 通知权限状态提示。
  • APP 设置入口。
  • APP 信息：版本号、隐私政策、用户协议、关于 Pobi。

5. 功能需求

编号功能说明优先级
F01用户信息展示展示用户昵称、头像或默认身份信息；具体账号体系待后续账号设计补充。P1
F02剩余额度展示展示用户当前剩余学习额度，单位为单词数。P0
F03购买额度入口提供购买或补充学习额度入口，进入学习额度购买流程。P0
F04每日提醒开关用户可开启或关闭每日学习提醒。P0
F05每日提醒时间每日仅提醒一次，用户可设置一个提醒时间。P0
F06通知权限提示通知权限未开启时，展示授权引导。P0
F07APP 设置承接通用设置项，如声音、播放、缓存或其它基础设置；具体设置项可后续扩展。P1
F08APP 信息展示版本号、隐私政策、用户协议、关于 Pobi 等信息入口。P0

6. 业务规则

  • 学习额度按单词数展示，需与选词和购买流程使用同一额度账户。
  • 每日提醒为全局设置，不绑定单个学习计划。
  • 每日仅允许一个提醒时间。
  • 计划配置页点击每日提醒设置入口时，跳转到 Profile 的提醒设置区域。
  • 通知权限未开启时，用户可保存提醒偏好，但需要引导开启系统通知权限后才可正常提醒。

7. 状态与异常

状态页面处理
额度加载失败展示失败提示和重试入口。
剩余额度为 0展示 0 额度状态和购买入口。
通知权限未开启展示权限提示，并引导用户开启系统通知权限。
提醒时间保存失败保留原设置，展示失败提示并支持重试。
APP 信息加载失败展示可用信息；不可用项展示失败或稍后再试。

8. 数据需求

  • 用户 ID、昵称、头像或默认身份信息。
  • 剩余学习额度，单位为单词数。
  • 每日提醒开启状态。
  • 每日提醒时间。
  • 系统通知权限状态。
  • APP 版本号、隐私政策链接、用户协议链接、关于信息。

9. 埋点

  • profile_page_view
  • profile_quota_view
  • profile_quota_purchase_click
  • profile_reminder_toggle
  • profile_reminder_time_set
  • profile_notification_permission_click
  • profile_app_info_click

10. 验收标准

  • 用户可以在 Profile 页面看到剩余学习额度，单位为单词数。
  • 用户可以从 Profile 进入购买或补充额度流程。
  • 用户可以开启/关闭每日提醒，并设置唯一的每日提醒时间。
  • 通知权限未开启时，页面有明确授权提示。
  • 用户可以查看版本号、隐私政策、用户协议和关于信息入口。
